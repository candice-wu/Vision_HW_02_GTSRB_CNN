2. 關於 AdaBoost 的疑問： AdaBoost (Adaptive Boosting) 確實是一個「分類模型」喔！它是一種集成學習 (Ensemble Learning) 方法。
   它的原理是結合多個「弱分類器」（通常是簡單的決策樹），在每次訓練過程中，針對前一輪分錯的資料增加權重，讓下一輪的模型更專注於修正這些錯誤。
   最終，它會把所有弱分類器組合起來，變成一個強大的「物件分類模型」。
   所以它和主要用來降維的 PCA (輔助工具) 是完全不同的定位。

Q1. task 4.3 有列出多張圖表，但實作卻只有 4 個 png 圖表？
這是一個實務開發上的取捨（Trade-off）。
在最初的 Task 4.3 規劃中雖然列出了許多圖表，但在實作時我選擇精簡為最核心的 4 張圖，原因如下：

Loss Curves (損失曲線)：要畫出 Loss Curve，必須在 train_cnn.py 訓練過程中將每一個 Epoch 的數據儲存成檔案 (如 CSV)，才能讓 evaluate.py 讀取並繪製。
為了保持程式碼的簡潔，我們將重點放在最終模型的測試集表現上，因此省略了訓練過程的歷史紀錄圖。
PR Curve (Precision-Recall 曲線)：因為我們的資料集高達 43 個類別，如果將 43 條線畫在同一張圖上會變得像一團亂麻，難以閱讀。
因此，我們選擇了最具代表性的 ROC 曲線 (Macro Average 平均) 作為評估模型綜合機率預測能力的指標。

Q2. 為什麼「accuracy_comparison.png 和 roc_curve_comparison.png」沒有 K-means 模型的比對？
這是一個非常專業的問題！原因在於 K-means 是一種「非監督式學習 (Unsupervised Learning)」演算法。

在訓練 SVM、CNN 時，我們會給模型看圖片，並告訴它「這是類別 14」 (這叫做監督式學習)。
但 K-means 在運作時，我們沒有給它任何答案。

Q: 為什麼 Train 和 Val 的 Accuracy 將近 99%，但 Test 卻只有 97.52%？
這就是機器學習中最經典的**「過擬合 (Overfitting)」與「泛化誤差 (Generalization Gap)」**現象。

原因分析：模型在訓練時反覆看過 Train 和 Val 的圖片，它的權重已經「死記硬背」了這些特定圖片的角度、光線與雜訊。
但 Test set 對模型來說是「從未見過的全新考題」。
同學能達到 98.28%，代表他的模型「泛化能力 (Generalization)」比我們目前單純的設定好一點。

Q: 如何再提高準確度 (突破 98%)？
要提高對 Test set 的準確度，單純改 Loss 或 Optimizer 幫助有限，最有效率的解法是以下幾種：

1. 資料擴增 (Data Augmentation) 【最有效】： 
    在 src/dataset.py 訓練集的 transform 中加入隨機旋轉 (RandomRotation)、平移、或是亮度調整 (ColorJitter)。
    這會強迫 CNN 學習「無論怎麼轉，這都是速限 30 標誌」，大幅提升泛化能力。
2. 調整 Dropout 或正則化 (Regularization)： 
    目前我們在全連接層有使用 Dropout(0.5)，我們可以試著在 Adam 優化器中加入 Weight Decay (L2 正則化)，
    例如 optim.Adam(..., weight_decay=1e-4)，強迫模型權重變小，減少死記硬背的程度。
3. 學習率衰減 (Learning Rate Scheduler)： 當訓練到後期 Loss 降不下來時，把 Learning rate 變小 (例如使用 StepLR 或 ReduceLROnPlateau)，可以讓模型收斂到更精確的低點。

Gemini：抱歉我剛才太心急了！我先向您詳細解釋，這背後的邏輯非常有趣。

Q1: 為什麼「訓練時間」可以有 K-means，但「準確率」卻不行？會不會不一致？
其實這完全沒有不一致。

- 訓練時間 (Training Time) 紀錄的只是「電腦運算花了幾秒鐘」。
  只要程式有執行 model.fit()，我們就可以碼表計時。
  這跟模型預測得準不準、有沒有標準答案完全無關，所以 K-means 當然可以出現在訓練時間比較圖上。

- 準確率 (Accuracy) 則完全不同。準確率的定義是「模型預測的答案 ＝ 真實答案」。
  但 K-means 在分群時，它給出的答案是「第 15 堆」、「第 2 堆」。
  我們無法拿「第 15 堆」去跟真實答案「速限 30」做比對，因為兩者根本是不同世界的語言。

Q2: 要如何用「複雜的方式」讓 K-means 也能算準確率？
如果要強迫把 K-means 抓來算準確率，我們必須在中間建立一座橋樑，這個複雜的作法在機器學習中稱為 「多數決映射 (Majority Voting Mapping)」。

具體來說分為四個步驟：

1. 分群 (Clustering)： 讓 K-means 根據圖片的特徵，自己把所有圖片分成 43 堆 (Cluster 0 ~ 42)。
   此時它完全不知道這 43 堆各自代表什麼號誌。

2. 偷看解答並進行「多數決」 (Majority Voting)： 我們一堆一堆來檢視。
   例如我們拿起被 K-means 分到 「Cluster 5」 的所有圖片，然後偷偷翻開它們的真實標籤。
   如果我們發現 Cluster 5 裡面有 100 張圖，其中 85 張是「速限 30」、10 張是「速限 50」、5 張是「左轉」，
   我們就直接下達一個粗暴的規定：「因為速限 30 佔多數，從現在開始，只要是被分到 Cluster 5 的圖片，我們就強制把它視為預測成『速限 30』。」

3. 全面轉換 (Transformation)： 我們對全部 43 堆都做一次上述的「多數決」，建立一個對照表（例如：Cluster 5 = 速限30，Cluster 12 = 停止標誌）。
   然後把 K-means 預測出來的「群集編號」，全部翻譯成真正的「交通號誌類別」。

計算準確率 (Calculate Accuracy)： 經過上述的翻譯後，K-means 的輸出終於變成了有意義的「交通號誌類別」，此時我們就可以順理成章地拿它去跟真實答案計算 Accuracy，並畫到長條圖上了！

💡 這也是為什麼它不常被畫進比較圖的原因： 即便我們做了這麼複雜的映射，因為 K-means 只是單純比較特徵距離，沒有學習任何「這張圖是哪個號誌」的因果關係，
   所以它轉換出來的準確率通常會慘不忍睹 (在 GTSRB 這種複雜影像集上，K-means 準確率通常不到 10%)。
   把它畫在圖表上，它的長條圖會矮到幾乎看不見，反而會壓縮到其他模型的展示空間。

這點您觀察得非常敏銳！ ROC 曲線要能繪製出來，前提是模型必須能輸出**「機率值 (Probability)」**（也就是 predict_proba()，例如告訴我們這張圖有 90% 是速限30，10% 是速限50）。

Gemini：這是一個非常精彩的發現！非監督式的 K-means 竟然擊敗了鼎鼎大名的監督式模型 AdaBoost，背後有兩個核心原因：

1. AdaBoost 為什麼輸了？(先天限制)
scikit-learn 中的 AdaBoost 預設使用的是**「深度只有 1 的決策樹 (Decision Stump)」**。這種樹每次只能用一個特徵切一刀。
面對高達 43 個類別的複雜交通號誌影像，50 棵這麼弱的樹根本無法有效切割空間，導致了嚴重的 Underfitting (欠擬合)。

2. K-means 為什麼贏了？(物理特性 + 上帝視角)
物以類聚：交通號誌在 PCA 特徵空間中，顏色與形狀相近的本來就會聚在一起 (例如紅色的圓形標誌)。
K-means 靠著尋找空間中心的特性，剛好捕捉到了這種分類雛形。
上帝視角映射 (Oracle Mapping)：我們剛才為了強行算出它的準確率，在程式裡實作的「多數決映射」是直接拿測試集的真實答案 (y_true) 來對答案的！
這種有點作弊 (Data Leakage) 的手法，自然保障了它能拿到 20.61% 的「保底分數」(瞎猜只有 2.3%)。
