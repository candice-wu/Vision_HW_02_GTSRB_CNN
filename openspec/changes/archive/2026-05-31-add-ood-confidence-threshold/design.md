## Context

在目前的系統架構下，Streamlit 應用程式 (`gtsrb.py`) 提供單張影像推論展示 (`tab1`)。當使用者點擊隨機抽樣或自行上傳圖片時，系統會調用已訓練好的深度學習 (CNN+STN) 以及 5 個機器學習模型 (SVM, RF, KNN, AdaBoost, NN) 進行即時推理。然而，當輸入是一張非德國交通號誌（例如台灣交通號誌或一般的貓狗圖片等分佈外 OOD 影像）時，由於模型分類器最後一層只會輸出 43 類的相對機率或距離，系統會被迫給出一個極具誤導性的高信心度錯誤預測。

因此，本設計旨在引入一個輕量級、非侵入式（無需重訓模型）的 **0.75 置信度安全門檻攔截決策機制**，在推理階段動態截斷低信賴度的預測，提升系統的安全防衛層次。

## Goals / Non-Goals

### Goals
- 在單張影像推理階段，動態提取各模型預測的最大機率值（信賴度）。
- 設定信賴度安全限制閥值為 `0.75` (75%)。
- 當最大機率值 `< 0.75` 時，攔截該預測，不顯示原分類 Class 數值，改為彈出優雅的黃橘色或紅色學術警告：`「這是非德國交通號誌」`（或「偵測到未知號誌/干擾」）。
- 確保此攔截機制不會影響測試集指標評估腳本 (`src/evaluate.py`) 與備份圖表之數據正確性。
- 實作對所有已導入模型的最大信賴度安全提取機制：
  - **CNN**：以 `torch.softmax` 取得 43 類機率，並取最大值。
  - **NN (MLPClassifier), RF, KNN, AdaBoost**：以 scikit-learn 的 `predict_proba()` 取得最大機率。
  - **SVM**：若未開啟 `probability=True`，則以 `decision_function` 的最大距離經由常規歸一化（如 Sigmoid）或設定對應閾值進行信心判定。

### Non-Goals
- 重新訓練現有已存檔的模型權重（不修改模型訓練核心代碼與模型檔本身，完全在推理階段實作）。
- 影響 `tab2` 與 `tab3` 的雙階段歷史指標與評估圖表展示。

## Decisions

### 1. 0.75 門檻安全過濾機制
- **決策**：在 `gtsrb.py` 的推論迴圈中，針對每一個模型獨立實作門檻檢查。
- **理由**：不同的模型對同一張 OOD 影像的反應不同，有些模型可能被騙，有些模型則能成功識別出低置信度。獨立檢查能完整呈現不同模型在遭遇干擾時的強健度差異，提供優異的學術對比價值。

### 2. SVM 邊界距離之信心分數轉換
- **決策**：若 SVM 模型未開啟 `predict_proba()`（即未設置 `probability=True`），我們可以使用決策函數 `decision_function(img_pca)[0]`。
  - 由於距離數值範圍較大，我們可以使用一個簡易的 Softmax 或 Sigmoid 函數將其對數距離映射為近似的機率；或者設定決策距離閾值（例如最大距離小於某個特定閥值時判定為 OOD）。
- **理由**：重新訓練 SVM 以啟用 `probability=True` 會耗費極大的運算資源與時間（Platt Scaling 會重複進行 K-fold 交叉驗證），採用推理端距離對數映射是最高效且優雅的學術妥協方案。

## Risks / Trade-offs

- **[Risk] 部分真實但品質極差的德國號誌被誤攔截** ➔ **[Mitigation]** 在 Web 介面上，提供明確的攔截說明。說明這是出於「安全防範（AI Safety）」的安全策略，寧可拒絕回答，也絕不給出錯誤預測，這是自駕車控制系統的標準規範。
- **[Risk] 傳統機器學習模型被大面積攔截** ➔ **[Mitigation]** 傳統 ML 因為沒有空間特徵卷積，對困難影像的信心度天生偏低。我們可以在 Streamlit 介面上以灰色字體提示其原始低信心度值，並清楚寫出攔截警告，這正好能向教授對比出「優化後 CNN」極度穩定且信心度高昂的強大優勢。
